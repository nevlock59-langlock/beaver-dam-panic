# Project Rules

- Explain in Korean.
- Build a cute mobile-first web minigame.
- Keep the first version simple and playable.
- Do not over-engineer.
- Prefer one App.jsx and one CSS file at first.
- Do not use real company/work data.
- Use only dummy game content.
- Before changing files, explain the plan.
- MVP first: start screen, game screen, result screen.
- First control scheme: tap material, then tap hole. Add drag-and-drop later only if MVP works.
