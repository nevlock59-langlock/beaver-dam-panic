# Beaver Dam Panic (비버 댐 수리 대작전) - 기획안

> **“홍수 직전의 댐을 구하라! 비버와 함께하는 댐 수리 대작전”**

---

## 1. 프로젝트 개요

**Beaver Dam Panic**은 홍수 직전의 댐을 비버와 함께 수리하는 웹 기반 미니게임입니다.  
사용자는 제한 시간 동안 댐에 생기는 구멍을 확인하고, 구멍 크기에 맞는 재료를 선택해 수리해야 합니다. 시간이 지날수록 수압이 상승하며, 수압이 100%에 도달하면 댐이 무너지고 게임이 종료됩니다.

- **장르:** 모바일 퍼스트 웹 미니게임 / 캐주얼 반응형 게임
- **플랫폼:** 웹 브라우저
- **조작 방식:** 모바일 터치 및 PC 마우스 클릭
- **주요 목표:** 제한 시간 동안 댐을 지켜내기
- **플레이 모드:** Easy 30초 / Hard 60초
- **배포 방식:** Vercel 웹 배포
- **버전관리:** GitHub

본 게임은 별도 설치 없이 URL 접속만으로 바로 플레이할 수 있으며, 모바일 세로 화면을 중심으로 설계하되 PC 브라우저에서도 플레이 가능하도록 구성했습니다.

---

## 2. 핵심 게임 플레이

Beaver Dam Panic의 핵심 플레이는 **“재료 선택 → 구멍 수리 → 수압 관리 → 성공/실패 판정”**으로 이루어집니다.

### 핵심 루프

1. 사용자가 Easy 또는 Hard 모드를 선택합니다.
2. 게임이 시작되면 댐에 구멍이 나타납니다.
3. 사용자는 구멍 크기에 맞는 재료를 선택합니다.
4. 선택한 재료로 구멍을 수리합니다.
5. 시간이 지날수록 수압과 난이도가 상승합니다.
6. 수압이 100%에 도달하면 실패합니다.
7. 제한 시간까지 버티면 성공합니다.
8. 결과 화면에서 점수, 별 평가, 최고 기록을 확인합니다.
9. 다시 도전하며 더 높은 점수에 도전합니다.

### 난이도 모드

- **Easy Mode:** 30초 생존 모드  
  처음 플레이하는 사용자가 규칙을 익히기 쉬운 기본 모드입니다.

- **Hard Mode:** 60초 생존 모드  
  더 긴 시간 동안 상승하는 수압을 버티는 도전형 모드입니다.

### 성공/실패 조건

- **성공:** 제한 시간까지 댐을 지켜내면 성공
- **실패:** 수압이 100%에 도달하면 댐 붕괴

## 3. 주요 기능 (Core Features)

### 튜토리얼

처음 접속한 사용자도 게임 규칙을 빠르게 이해할 수 있도록 **How to Play 모달**을 제공합니다.  
재료 선택, 수압 상승, Easy/Hard 생존 조건을 짧고 직관적으로 안내합니다.

### 난이도 선택

시작 화면에서 Easy / Hard 모드를 선택할 수 있습니다.  
Easy는 30초, Hard는 60초로 구성해 초보자와 도전형 플레이어 모두에게 맞는 플레이 흐름을 제공합니다.

### 일시정지

게임 도중 잠시 멈출 수 있는 Pause 기능을 제공합니다.  
사용자는 게임 중간에 흐름을 멈추고 다시 이어서 플레이할 수 있습니다.

### 기록 시스템

트로피 버튼을 통해 최고 기록을 확인할 수 있습니다.  
기록 화면은 복잡한 통계표 대신 **전체 최고점 Top 1과 해당 모드**만 표시해 직관성을 높였습니다.

### 결과 피드백

게임 종료 후 성공/실패 여부, 최종 점수, 별 평가, 최고 기록 여부를 제공합니다.  
결과 화면은 단순 종료 화면이 아니라, 사용자가 다시 도전하고 싶도록 만드는 피드백 화면으로 설계했습니다.

### 모바일 최적화

모바일 세로 화면을 기준으로 설계했으며, 사용자가 가로모드로 전환할 경우 세로모드 권장 안내 오버레이를 제공합니다.

## 5. AI 협업 워크플로 (AI-assisted Vibe Coding)

본 프로젝트는 기획부터 배포까지 **AI-assisted vibe coding** 방식으로 제작되었습니다.  
AI를 단순 코드 생성 도구가 아니라, 빠른 프로토타입 제작을 위한 협업 도구로 활용했습니다.

### 활용 방식

- **ChatGPT Plus:** 게임 기획, 요구사항 구체화, 기능 우선순위 판단, QA 체크리스트 도출
- **Gemini CLI on Termux:** 모바일 환경에서 코드 구현 및 수정
- **GitHub:** 변경사항 저장 및 버전관리
- **Vercel:** 클라우드 배포
- **모바일/PC 테스트:** 실제 플레이 환경에서 UI와 조작감 확인

### 제작 흐름

1. 기능 아이디어 도출
2. 요구사항을 자연어 프롬프트로 정리
3. Gemini CLI를 통한 코드 수정
4. 모바일 localhost 및 배포 URL에서 테스트
5. UI/UX 확인 후 승인 또는 수정
6. GitHub 커밋
7. Vercel 자동 배포

이 과정에서 최종 판단, 기능 범위 조정, 사용성 검수, 제출 안정성 판단은 사람이 직접 수행했습니다.  
AI는 구현 속도를 높이는 도구로 활용했고, 게임의 방향성과 완성도 판단은 human-in-the-loop 방식으로 진행했습니다.

## 6. 프로젝트의 차별점

1. **설치가 필요 없는 웹 접근성**  
   URL 접속만으로 모바일과 PC에서 즉시 플레이할 수 있습니다.

2. **짧고 명확한 플레이 구조**  
   Easy 30초, Hard 60초 구조로 한 판의 부담을 낮추고 반복 플레이를 유도했습니다.

3. **친절한 온보딩**  
   How to Play 모달, 직관적인 시작 화면, 결과 피드백을 통해 처음 플레이하는 사용자도 빠르게 적응할 수 있습니다.

4. **모바일 UX 보강**  
   세로모드 권장 안내, 큰 터치 버튼, 단순한 조작 구조를 통해 모바일 플레이 환경을 고려했습니다.

5. **기록과 도전 요소**  
   최고 기록, 별 평가, Easy/Hard 모드를 통해 단순한 1회성 플레이가 아니라 반복 도전을 유도합니다.

6. **저비용·고속 프로토타이핑**  
   AI 도구와 무료 배포 인프라를 활용해 짧은 시간 안에 기획, 구현, 테스트, 배포까지 완료했습니다.

## 7. 평가 기준 대응

### 참신성

Beaver Dam Panic은 “댐 수리”라는 직관적인 상황을 짧은 반응형 미니게임으로 구성했습니다.  
사용자는 단순히 점수를 얻는 것이 아니라, 물이 차오르는 상황에서 구멍 크기에 맞는 재료를 빠르게 선택해야 합니다. 귀여운 비버 캐릭터와 홍수 직전이라는 상황을 결합해 짧은 시간 안에 이해되는 독자적인 콘셉트를 만들었습니다.

### 완성도 및 안정성

게임은 웹 브라우저에서 별도 설치 없이 실행되며, GitHub와 Vercel을 통해 배포되었습니다.  
Easy/Hard 모드, 일시정지, 결과 화면, 기록 확인, 세로모드 안내 등 기본적인 플레이 흐름에 필요한 기능을 포함했습니다. 모바일과 PC 환경에서 직접 테스트하며 주요 화면이 정상적으로 동작하도록 구성했습니다.

### 사용성 및 UI/UX

처음 접속한 사용자도 게임 방법을 이해할 수 있도록 How to Play 모달을 제공합니다.  
시작 화면에서는 난이도 선택, 기록 확인, 게임 시작 버튼을 한 화면에서 확인할 수 있으며, 플레이 화면에서는 점수, 시간, 수압, 선택 재료를 직관적으로 보여줍니다. 모바일 세로 화면을 기준으로 설계하고, 가로모드에서는 세로모드 권장 안내를 제공합니다.

### 기획·구현 완성도

핵심 게임 루프는 “재료 선택 → 구멍 수리 → 수압 상승 → 성공/실패 판정 → 기록 확인 → 재도전”으로 구성했습니다.  
Easy와 Hard 모드를 분리해 초보자와 도전형 플레이어 모두에게 맞는 플레이 흐름을 제공합니다. 결과 화면의 별 평가와 최고 기록은 반복 플레이를 유도합니다.

### 재미 및 몰입도

짧은 제한 시간, 상승하는 수압, 빠른 재료 선택이 결합되어 간단하지만 긴장감 있는 플레이를 제공합니다.  
한 판의 플레이 시간이 짧아 부담 없이 재도전할 수 있으며, Hard 모드와 최고 기록 시스템을 통해 더 높은 점수를 노리는 동기를 제공합니다.
