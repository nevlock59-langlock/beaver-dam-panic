# Beaver Dam Panic - Game Design Document

## 1. 프로젝트 개요

**Beaver Dam Panic**은 홍수 직전의 댐을 비버와 함께 수리하는 웹 미니게임입니다.  
사용자는 제한 시간 동안 구멍 크기에 맞는 재료를 선택해 댐을 고치고, 시간이 지날수록 강해지는 수압을 버티며 생존에 도전합니다.

- **장르:** 웹 미니게임 / 캐주얼 반응형 게임
- **플랫폼:** 웹 브라우저
- **조작 방식:** 마우스 클릭 / 모바일 터치
- **주요 플레이 시간:** Easy 30초, Hard 60초
- **핵심 목표:** 댐이 무너지기 전까지 제한 시간 생존
- **배포 방식:** Vercel 웹 배포
- **저장소:** GitHub

---

## 2. 기획 의도

이 게임은 짧은 시간 안에 규칙을 이해하고 바로 플레이할 수 있는 “10분 내 완결형 웹 미니게임”을 목표로 기획되었습니다.

복잡한 조작이나 긴 설명 없이도 사용자가 다음 흐름을 직관적으로 이해할 수 있도록 구성했습니다.

1. 재료를 고른다.
2. 댐의 구멍을 수리한다.
3. 수압이 올라가는 긴장감을 버틴다.
4. 제한 시간까지 생존하면 성공한다.
5. 기록과 별 평가를 보고 다시 도전한다.

핵심 재미는 “빠른 판단과 반복 도전”입니다.  
구멍 크기에 맞는 재료를 고르고, 수압이 높아지는 상황에서 당황하지 않고 대응하는 짧은 긴장감을 제공하고자 했습니다.

---

## 3. 타깃 사용자 및 플레이 환경

### 타깃 사용자

- 짧고 가벼운 웹게임을 선호하는 사용자
- 모바일에서 바로 실행 가능한 게임을 원하는 사용자
- 별도 설치 없이 링크만으로 플레이하고 싶은 사용자
- 귀여운 그래픽과 단순한 조작을 선호하는 캐주얼 플레이어

### 플레이 환경

- 모바일 브라우저
- PC 브라우저
- 터치 입력 및 마우스 입력 모두 지원

게임은 모바일 세로 화면을 중심으로 설계했지만, PC 브라우저에서도 플레이할 수 있도록 반응형 웹 형태로 구성했습니다.

---

## 4. 핵심 게임 루프

Beaver Dam Panic의 기본 플레이 루프는 다음과 같습니다.

1. 사용자가 Easy 또는 Hard 모드를 선택한다.
2. 게임이 시작되면 댐에 구멍이 나타난다.
3. 사용자는 구멍 크기에 맞는 재료를 선택한다.
4. 선택한 재료로 구멍을 수리한다.
5. 시간이 지날수록 수압과 난이도가 상승한다.
6. 수압이 100%에 도달하면 실패한다.
7. 제한 시간까지 버티면 성공한다.
8. 결과 화면에서 점수, 별 평가, 최고 기록을 확인한다.
9. 다시 플레이하며 더 높은 점수에 도전한다.

---

## 5. 게임 규칙

### 기본 목표

댐이 무너지기 전에 제한 시간 동안 최대한 많은 구멍을 수리하고 생존합니다.

### 실패 조건

- 수압이 100%에 도달하면 댐이 무너지고 게임이 종료됩니다.

### 성공 조건

- Easy 모드: 30초 동안 생존
- Hard 모드: 60초 동안 생존

### 재료 선택

사용자는 화면 하단의 재료 버튼을 선택해 댐의 구멍을 수리합니다.  
구멍 크기와 상황에 맞는 재료를 빠르게 선택하는 것이 핵심입니다.

예시 재료:

- Leaf
- Wood
- Stone

### 난이도 증가

게임이 진행될수록 수압과 위험도가 증가합니다.  
초반에는 사용자가 규칙을 익힐 수 있도록 비교적 안정적이지만, 시간이 지날수록 더 빠른 판단과 대응이 필요해집니다.

---

## 6. 난이도 모드

### Easy Mode

Easy 모드는 기본 플레이 모드입니다.

- 제한 시간: 30초
- 짧고 빠르게 플레이 가능
- 처음 플레이하는 사용자에게 적합
- 기본 규칙과 조작을 익히기 쉬운 모드

### Hard Mode

Hard 모드는 도전용 모드입니다.

- 제한 시간: 60초
- Easy보다 긴 생존 시간
- 시간이 지날수록 더 높은 긴장감 제공
- 고득점과 기록 갱신을 목표로 하는 모드

Hard 모드는 단순히 시간을 늘리는 것에 그치지 않고, 장기 생존에 따른 긴장감을 제공하도록 설계했습니다.

---

## 7. 점수 및 기록 시스템

게임은 플레이 결과에 따라 점수를 제공합니다.  
사용자는 결과 화면에서 최종 점수와 최고 기록 여부를 확인할 수 있습니다.

### 점수 설계 의도

- 수리를 많이 할수록 점수 상승
- 오래 버틸수록 높은 점수 가능
- Hard 모드는 더 긴 플레이와 높은 난이도를 통해 도전 욕구 제공

### 최고 기록

트로피 버튼을 통해 현재까지의 최고 기록을 확인할 수 있습니다.  
기록 화면은 복잡한 통계표가 아니라 전체 최고점 1개와 해당 모드를 보여주는 방식으로 단순화했습니다.

예시:

- Best Record
- 최고 점수
- 해당 기록이 나온 모드: EASY MODE 또는 HARD MODE

---

## 8. 별 평가 시스템

결과 화면에서는 생존 시간에 따라 별 평가를 제공합니다.  
별은 사용자가 다음 플레이에서 더 높은 목표를 세울 수 있도록 돕는 간단한 피드백 장치입니다.

### Easy Mode 기준

- 10초 이상: 별 1개
- 20초 이상: 별 2개
- 30초 생존: 별 3개

### Hard Mode 기준

- 20초 이상: 별 1개
- 40초 이상: 별 2개
- 60초 생존: 별 3개

별 평가는 게임의 성공/실패 여부와 별도로, 사용자의 진행도를 직관적으로 보여주는 역할을 합니다.

## 9. 주요 기능

### 9.1 시작 화면

시작 화면에서는 게임의 전체 구조를 간단히 파악할 수 있습니다.

- 게임 제목
- 비버 캐릭터 그래픽
- How to Play 버튼
- Records 버튼
- Easy / Hard 모드 선택
- 수리 시작 버튼
- 제작 크레딧

### 9.2 How to Play

처음 플레이하는 사용자를 위해 정적 튜토리얼 모달을 제공합니다.

튜토리얼은 다음 내용을 짧게 안내합니다.

- 재료를 골라 댐의 구멍을 막는다.
- 시간이 지날수록 물살이 강해진다.
- Easy는 30초, Hard는 60초 동안 댐을 지키면 승리한다.

### 9.3 플레이 화면

플레이 화면에서는 현재 상태를 즉시 파악할 수 있도록 정보를 배치했습니다.

- 현재 모드
- 점수
- 남은 시간
- 수압
- 선택한 재료
- 재료 선택 버튼
- 일시정지 버튼
- 댐 수리 영역

### 9.4 일시정지

게임 도중 일시정지 기능을 제공합니다.  
사용자는 잠시 게임을 멈추고 다시 이어서 플레이할 수 있습니다.

### 9.5 결과 화면

게임 종료 후 결과 화면에서 플레이 결과를 확인할 수 있습니다.

- 성공 또는 실패 여부
- 최종 점수
- 최고 기록 여부
- 별 평가
- 재도전 버튼
- 결과 피드백 문구

결과 화면은 단순히 종료를 알리는 화면이 아니라, 사용자가 다음 플레이를 시도하도록 유도하는 역할을 합니다.

### 9.6 세로모드 권장 안내

게임은 모바일 세로 화면을 기준으로 설계되었습니다.  
사용자가 모바일에서 가로모드로 전환할 경우, 세로모드 권장 오버레이를 보여줍니다.

이 기능은 모바일 화면에서 레이아웃이 깨지는 상황을 방지하고, 사용자에게 권장 플레이 환경을 명확히 안내하기 위해 추가했습니다.

## 10. UI/UX 설계 포인트

### 직관적인 조작

재료 선택과 구멍 수리라는 단순한 구조를 통해 사용자가 빠르게 규칙을 이해할 수 있도록 했습니다.

### 짧은 플레이 시간

Easy 30초, Hard 60초 구조를 통해 한 판의 부담을 낮추고 반복 플레이를 유도했습니다.

### 모바일 중심 설계

모바일 세로 화면을 기준으로 레이아웃을 구성했으며, 터치 조작에 적합하도록 버튼과 정보 영역을 배치했습니다.

### 귀여운 시각 디자인

비버, 댐, 수압, 재료 등의 요소를 밝고 귀여운 그래픽 톤으로 표현해 캐주얼 미니게임의 접근성을 높였습니다.

### 피드백 중심 결과 화면

성공/실패, 별 평가, 최고 기록, 재도전 안내를 결과 화면에 배치해 사용자가 자신의 플레이 결과를 쉽게 이해할 수 있도록 했습니다.

## 12. AI-assisted 개발 워크플로

본 프로젝트는 AI-assisted vibe coding 방식으로 제작되었습니다.

ChatGPT Plus는 기획 정리, 요구사항 구체화, 기능 우선순위 판단, QA 체크리스트 도출에 활용했습니다.  
Gemini CLI on Termux는 모바일 환경에서 실제 코드 수정과 구현을 진행하는 도구로 활용했습니다.  
이후 GitHub를 통해 변경사항을 관리하고, Vercel을 통해 웹 브라우저에서 즉시 실행 가능한 형태로 배포했습니다.

제작 과정은 다음과 같은 반복 구조로 진행되었습니다.

1. 기능 아이디어 도출
2. 요구사항을 자연어 프롬프트로 정리
3. Gemini CLI를 통한 코드 수정
4. 모바일 localhost 또는 배포 URL에서 테스트
5. UI/UX 확인 후 승인 또는 수정
6. GitHub 커밋
7. Vercel 자동 배포

이 과정에서 AI는 단순한 코드 생성 도구가 아니라 빠른 프로토타입 제작을 위한 협업 도구로 활용되었습니다.  
최종 판단, 기능 범위 조정, 사용성 검수, 제출 안정성 판단은 사람이 직접 수행했습니다.

## 13. 제작 및 운영 관점의 특징

### 빠른 프로토타이핑

짧은 시간 안에 게임 아이디어를 실제 플레이 가능한 웹앱으로 구현했습니다.

### 낮은 제작 비용

기존 구독 도구와 저비용 AI API 사용, 무료 GitHub/Vercel 배포 환경을 활용해 낮은 비용으로 배포 가능한 결과물을 만들었습니다.

### 버전관리 기반 개발

기능 추가 후 중간 저장을 반복하며 GitHub에 커밋했습니다.  
이를 통해 개발 중 문제가 발생하더라도 안정적인 지점으로 돌아갈 수 있도록 했습니다.

### 실기기 테스트

모바일과 PC에서 직접 플레이하며 조작감, 화면 배치, 가독성, 결과 화면을 확인했습니다.

## 14. 차별화 포인트

1. 모바일-first 웹 미니게임
   - 설치 없이 링크만으로 플레이 가능
   - 모바일 세로 화면 기준 UX 제공
   - PC 브라우저에서도 플레이 가능

2. 짧고 반복 가능한 게임 구조
   - Easy 30초 / Hard 60초
   - 한 판의 부담이 작고 재도전이 쉬움

3. 친절한 온보딩
   - How to Play 모달
   - 직관적인 시작 화면
   - 결과 화면 피드백

4. 완성도 보강 기능
   - 일시정지
   - 기록 모달
   - 세로모드 안내
   - 별 평가
   - 최고 기록 표시

5. AI-assisted 제작 방식
   - 모바일 환경에서 AI 코딩 도구를 활용해 기획, 구현, 테스트, 배포까지 진행

## 15. 향후 확장 아이디어

### 추가 스테이지

비 오는 날, 폭풍우, 야간 수리 등 다양한 테마의 스테이지를 추가할 수 있습니다.

### 아이템 시스템

망치, 모래주머니, 긴급 배수 장치 등 일회성 아이템을 추가해 전략성을 높일 수 있습니다.

### 캐릭터 커스터마이징

비버 스킨, 댐 장식, 배경 테마 등을 추가해 수집 요소를 만들 수 있습니다.

### 랭킹 시스템

현재는 로컬 최고 기록 중심이지만, 향후 온라인 랭킹이나 공유 가능한 기록 카드를 추가할 수 있습니다.

### 사운드 효과

수리 성공, 수압 상승, 게임 성공/실패에 맞는 짧은 효과음을 추가하면 몰입도를 높일 수 있습니다.

구멍 크기에 맞는 재료 선택, 상승하는 수압, Easy/Hard 모드, 기록과 별 평가를 통해 단순하지만 명확한 재미를 제공합니다.

AI-assisted vibe coding workflow를 활용해 기획, 구현, 테스트, 배포까지 빠르게 진행한 점도 본 프로젝트의 주요 특징입니다.
